# ReAct

`Reason` + `Act` 核心思想是模仿人类解决问题，使用 `思考-行动-观察` 的循环

每一步的输出都遵循这一个特定的轨迹：
- `Thought`
    - 分析当前情况，分解任务
    - 指定下一步行动或者反思上一步结果
- `Action`
    - 智能体决定采取的具体行动
    - 调用外部工具
- `Observation`
    - 执行 `Action` 之后获取的返回结果

### 特点

##### 高解释性

通过 `Thought` 显示当前思考，使用户能够理解智能体的决策过程

##### 动态规划和纠错

通过 `Observation` 反馈，动态的调整智能体的行动，实现对任务的动态规划和纠错

##### 工具协同

将外部工具和大模型的输出进行协同，突破知识的时效性

### 局限

##### 对 LLM 的强依赖

高度依赖底层 LLM  的能力
- 推理
- 指令遵循
- 格式化输出

##### 执行效率

一次任务会涉及多次循环调用，对于多步骤的复杂任务，会需要很多时间和费用

##### 提示词设计

整个机制建立在提示词模板上，如果模板产生变化可能导致智能体的行动产生变化
